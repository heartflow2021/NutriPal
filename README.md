# 營養好朋友 NutriPal - 產品需求文檔 (PRD)

## 一、產品簡介

### 產品名稱
營養好朋友 NutriPal

### 產品願景
成為使用者最信賴的保健品推薦夥伴，透過個人化建議和專業知識，幫助人們做出更明智的健康決策。

### 產品目標
提供一個中文化、對話式互動的網站，幫助使用者依據自身健康需求與預算，快速推薦適合的 iHerb 保健品組合，同時可導流至聯盟行銷獲得被動收入。

### 產品價值主張
1. **個人化推薦**：根據用戶的健康需求、生活習慣和預算提供量身定制的保健品推薦
2. **專業指導**：將複雜的營養知識轉化為簡單易懂的個人建議
3. **省時省力**：節省用戶搜索和比較產品的時間
4. **可信賴資源**：提供準確的保健品資訊和使用建議

## 二、使用者需求與痛點

### 目標族群
- **主要目標**：30～50 歲身心健康關注者
- **消費特徵**：常購買保健品但感到選擇困難
- **健康關注**：有特定健康訴求（如睡眠、骨質、焦慮、視力等）
- **資訊行為**：具備初步保健知識，喜歡查資料，但缺乏判斷力

### 使用者痛點
1. **選擇障礙**：保健品選擇過多，難以篩選適合自己需求的產品
2. **知識缺口**：看不懂營養成分與機轉說明
3. **使用困惑**：不知道服用時機、劑量與搭配方式
4. **價值疑慮**：擔心花冤枉錢買錯產品
5. **資訊過載**：網路資訊雜亂，難以找到值得信賴的建議

### 使用者故事
- 作為一位常感到疲勞的上班族，我想找到適合我的能量補充保健品，以便能在工作時保持精力充沛。
- 作為一位關注關節健康的中年人，我想知道哪些產品對關節健康有幫助，以便能減輕關節不適。
- 作為一位預算有限的消費者，我想找到性價比高的保健品組合，以便能在有限預算內獲得最大健康效益。

## 三、產品核心功能（MVP）

### 1. 對話式推薦流程
- **啟動畫面引導**：「告訴我你的健康需求，我幫你配出最適合的保健品組合」
- **互動流程**：
  - 問題 1：目前最想改善的健康狀況（選項：精力提升、睡眠品質、腸胃健康、免疫力強化、關節靈活度、心血管健康、眼睛健康、壓力管理、其他）
  - 問題 2：平常生活型態（選項：辦公室工作者、經常運動、經常出差、照顧家庭、學生、其他）
  - 問題 3：預算範圍（選項：經濟型（$500-1000）、平衡型（$1000-2000）、完整型（$2000+））
  - 問題 4：服用頻率與接受度（選項：願意每日多次服用、僅能接受早晚一次、希望越簡單越好）
  - 問題 5：過往使用經驗與藥物過敏史（選填）
- **結果呈現**：完成問答後顯示個人化的保健品推薦組合

### 2. 保健品推薦清單
- **產品資訊卡片**，每項產品顯示：
  - 產品圖片（來自 iHerb）
  - 中文品名
  - 功效摘要（2-3 行）
  - 使用建議（如：早餐後／睡前服用）
  - 價格資訊（含折扣）
  - 為何推薦理由（針對用戶需求）
  - 點擊直達 iHerb 商品頁面的按鈕（含 affiliate code）
- **組合效益說明**：解釋該組合如何協同工作改善用戶的健康問題
- **替代選項**：預算更高或更低的替代產品建議

### 3. 個人化保健規劃
- **每日服用建議時間表**：視覺化呈現每日服用時間和注意事項
- **購買優先順序建議**：若預算有限，建議優先購買哪些產品
- **PDF 下載功能**：可下載或透過 email 獲取完整建議
- **服用提醒設置**：選擇是否接收 email 提醒

### 4. 網站整體設計
- **首頁**：
  - 清晰的價值主張與服務介紹
  - 使用案例展示（3-4個常見健康需求的推薦範例）
  - 如何使用的簡易步驟說明
  - 開始諮詢的明確入口按鈕
  - 信任元素（如專業背景說明、用戶見證）
- **互動頁**：
  - 嵌入對話模組，清晰的進度指示
  - 回答內容摘要，允許修改答案
  - 結果頁面的社群分享功能
  - 返回修改答案的選項

### 5. 內容支援功能
- **保健小知識庫**：常見保健問題的簡短解答
- **產品常見問題**：針對推薦產品的使用疑問解答
- **基礎營養知識**：幫助用戶理解推薦背後的科學原理

## 四、非功能性需求

### 用戶體驗需求
- **回應時間**：頁面載入時間不超過 3 秒
- **操作簡易性**：完成推薦流程平均需時不超過 3 分鐘
- **視覺設計**：清新專業的醫療健康風格，建立信任感
- **無障礙設計**：符合 WCAG 2.1 AA 級標準

### 技術需求
- **響應式設計**：在桌面、平板和手機上都能完美展現（RWD）
- **瀏覽器兼容性**：支援 Chrome、Safari、Firefox、Edge 最新版本
- **輕量快速載入**：優化 SEO 和用戶體驗
- **系統擴展性**：架構設計考慮未來接入 GPT API 或其他 AI 功能

### 安全與隱私
- **資料處理**：明確的隱私政策，說明如何使用用戶提供的健康資訊
- **資料保存**：非必要不儲存敏感健康資訊
- **第三方整合**：透明說明與 iHerb 等合作夥伴的資料分享政策

### 成本控制
- **開發資源**：
  - 前端網站：使用 Webflow 或 React + Next.js
  - 對話式互動：Landbot 或自建 React 元件
  - 主機：Vercel/Netlify（靜態網站託管）
- **營運成本**：
  - 內容更新：每月更新產品資訊和價格
  - 技術維護：低維護需求設計

### 長期（12+個月）

2. **多語言支持**：擴展至其他亞洲市場

## 六、技術架構（MVP階段）

### 前端架構
- **框架選擇**：React + Next.js（考慮 SEO 和靜態生成的優勢）
- **樣式方案**：Tailwind CSS + 自定義設計系統
- **狀態管理**：React Context 或 Redux（視複雜度而定）

### 後端服務
- **API 服務**：無伺服器函數（Vercel Functions/AWS Lambda）
- **資料存儲**：
  - 產品資料：Airtable 或 Supabase
  - 用戶互動資料：Firebase/Supabase
- **第三方整合**：
  - Email 服務：SendGrid/Mailchimp
  - 聯盟行銷追蹤：iHerb API 或自建追蹤系統

### 部署與運維
- **CI/CD**：GitHub Actions + Vercel 自動部署
- **監控**：基本的頁面分析和錯誤追蹤
- **效能優化**：圖片優化、代碼分割、緩存策略

## 七、成功指標與評估

### 關鍵績效指標（KPIs）
1. **流量指標**：
   - 每月訪問用戶數
   - 平均停留時間和跳出率
   - 完成推薦流程的轉化率

2. **業務指標**：
   - 通過 affiliate 連結的點擊率
   - 轉化為購買的比例
   - 平均訂單價值
   - 每位用戶產生的收入

3. **用戶參與指標**：
   - 保存/分享推薦結果的比例
   - 電子郵件訂閱率和打開率
   - 回訪用戶比例

### 成功標準（MVP階段）
- **月活躍用戶**：1,000+
- **推薦完成率**：開始流程的用戶中至少 60% 完成並查看結果
- **點擊轉化率**：至少 30% 的用戶點擊 affiliate 連結
- **用戶滿意度**：通過簡短問卷調查測量，目標滿意度 80%+

## 九、風險評估與緩解策略

### 潛在風險
1. **推薦準確性**：靜態推薦邏輯可能無法滿足所有用戶需求
   - **緩解策略**：明確說明推薦基於一般原則，不構成醫療建議；提供修改選項

2. **合規風險**：保健品推薦可能涉及健康聲明的法律限制
   - **緩解策略**：法律審查所有內容；加入免責聲明；避免做出疾病治療聲明

3. **競爭風險**：市場上可能出現類似服務
   - **緩解策略**：專注於中文用戶體驗和本地化內容；建立用戶社群和品牌忠誠度

4. **技術實現挑戰**：對話式推薦系統的複雜性
   - **緩解策略**：先使用簡化的決策樹邏輯，逐步引入更複雜的推薦算法
