📋 專案簡介
營養好朋友 (NutriPal) 是一個專為華人設計的健康生活知識平台，透過對話式互動，根據使用者的健康興趣、生活習慣和預算考量，整理並提供熱門的保健相關知識與資訊。
我們的使命是讓每個人都能輕鬆找到適合自己參考的健康知識，不需要成為專家也能獲取清晰的營養相關資訊。

🎯 核心功能
健康知識整理：根據健康興趣、生活習慣和預算整理相關營養知識
對話式體驗：友善的互動介面，像與朋友聊天般輕鬆獲取健康知識

## 知識庫詳細規劃

知識庫是 NutriPal 的核心資訊中心，專注於符合台灣衛福部規範的營養知識分享。

### 內容結構與來源
- **內容管理系統**：使用 Google Sheet 作為內容來源，包含文章標題和內容
- **自動化導入**：透過API自動將新內容從 Google Sheet 匯入至網站
- **內容分類**：
  - 營養素認識：不同營養素的基本特性與科學認識
  - 成分介紹：客觀描述保健品常見成分的特性與來源
  - 攝取資訊：國際組織對各類營養素的每日參考攝取量整理
  - 選購指南：客觀比較不同成分的選購要點
  - 常見問答：用戶經常詢問的健康知識問題

### 合規設計
- **符合衛福部規範**：嚴格避免療效導向詞彙，如「改善」、「預防」、「治療」等
- **專業審核機制**：所有文章經專業營養師審核確保科學性與合規性
- **資訊透明度**：清楚標示資料來源與參考文獻
- **明確免責聲明**：強調內容僅供參考，不構成醫療建議
- **合規用詞指南**：
  - 避免使用：「改善」、「增強」、「預防」、「治療」、「舒緩」等療效導向詞彙
  - 替代用詞：「營養素」、「成分介紹」、「使用資訊」、「營養知識」等

### SEO優化策略
- **語意化HTML結構**：使用合適的標題層級、section、article等標籤
- **Schema.org標記**：實現結構化資料標記，提升搜索引擎對內容的理解
- **關鍵字研究**：基於華人使用者搜尋習慣的關鍵字佈局
- **麵包屑導航**：優化網站結構與用戶體驗
- **行動裝置優化**：確保在各種設備上的閱讀體驗一致
- **Meta標籤優化**：為每個頁面提供精準的標題、描述和關鍵字
- **圖片優化**：使用適當的alt屬性、壓縮圖片大小

### 互動與用戶參與
- **內容收藏功能**：用戶可收藏感興趣的文章
- **知識測驗**：互動式營養知識測驗增強記憶與參與
- **推薦閱讀系統**：根據用戶關注的健康主題推薦相關文章
- **內容評分機制**：允許用戶為知識內容提供有用度評分
- **知識分享功能**：方便用戶將有用資訊分享到社交媒體

## 合規聲明與保障
NutriPal 為健康生活知識平台，嚴格遵守台灣衛福部相關規範：

- **知識性質**：本平台提供的內容僅為知識性資訊整理，不宣稱或暗示任何療效
- **聯盟行銷透明**：所有含有聯盟行銷連結的內容均明確標示
- **不提供醫療建議**：平台內容不構成醫療診斷、治療或預防疾病的建議
- **免責聲明**：使用者應在專業醫療人員指導下做出健康決策
- **資料來源標示**：所有內容均附有參考來源，確保資訊可追溯性

## 資訊整理算法詳情

NutriPal的資訊整理演算法是一個智能推薦系統，根據用戶的健康興趣、生活型態和預算考量，從營養知識資料庫中篩選出最適合的內容組合。算法的主要功能包括：

1. **基於主題的篩選**：根據用戶選擇的健康主題（如睡眠知識、免疫系統知識等）篩選相關資訊。

2. **生活型態匹配**：考慮用戶的生活習慣（如長時間工作、經常運動等），找出更適合的知識內容。

3. **預算優化**：根據用戶設定的預算範圍，使用貪婪算法選擇最合適的資訊組合，在預算內提供最佳效益。

4. **多維度排序**：綜合考慮資料評分、價格和熱門程度等因素，使用加權排序算法找出最適合用戶的內容。

5. **相關內容推薦**：基於已選內容的特性，分析並推薦相關的互補資訊，提供更全面的健康知識。

算法實現使用現代JavaScript，採用模塊化設計，包含數據緩存機制以提高性能。未來將加入機器學習組件，根據用戶反饋不斷優化推薦結果。

## Claude Task Master
本專案使用 Claude Task Master 進行任務管理和開發追蹤。Task Master 是一個 AI 驅動的任務管理系統，可以幫助開發團隊更有效地組織和實施開發計劃。

### 在 Cursor 中使用 Task Master
1. **初始化 Task Master**：在 Cursor 聊天視窗中輸入 `Can you please initialize taskmaster-ai into my project?`
2. **解析產品需求文檔**：輸入 `Can you parse my PRD at scripts/prd.txt?`
3. **查看任務列表**：輸入 `task-master list` 或 `What tasks do we have?`
4. **獲取下一個要執行的任務**：輸入 `What's the next task I should work on?`
5. **實現具體任務**：輸入 `Can you help me implement task <任務編號>?`
6. **展開複雜任務**：輸入 `Can you help me expand task <任務編號>?`

### 常用指令參考
- `task-master list`：列出所有任務
- `task-master next`：獲取下一個要執行的任務
- `task-master complete --id=<任務編號>`：標記任務為已完成
- `task-master expand --id=<任務編號>`：將任務分解為子任務
- `task-master update "<更新描述>"`：更新任務定義
- `task-master generate`：重新生成任務文件
